# T004 — UI cleanup

**Status:** planned  
**Maps to:** US-05 (app at a glance), US-06 (readable labels)  
**Depends on:** [T003 — Flutter Web + Playwright](T003.md)

## Goal

A small visual-polish pass over the existing DriftID web UI from T003. Tighten layout, spacing, and styling so the app reads as clean, flat, and consistent — without changing behavior, the API contract, or Playwright selectors. Also switch the Playwright demo recording to a standard iPhone screen size so the clip reads as a phone-shaped demo.

## Approach

Refine the existing widgets in `ui/lib/` only. No new screens, no new dependencies, no functional changes. Keep all `Key`s and `Semantics` labels intact so the T003 demo spec still passes. Follow the **simple and flat** UI guidelines in `AGENTS.md`: solid colors, clear typography, minimal chrome — no gradients, glass, heavy shadows, or decorative animation.

For the demo recording, update `ui/playwright.config.ts` to use a standard iPhone viewport (e.g. iPhone 14/13 — `390 × 844`) instead of the current `1280 × 720` desktop size, and match the recorded `video.size` to it so the WebM is phone-shaped.

### Proposed files

| File | Purpose |
|------|---------|
| `ui/lib/main.dart` | Centralize theme tweaks (flat `AppBar`, consistent component themes) |
| `ui/lib/screens/home_screen.dart` | Consistent spacing scale, flatter `AppBar`, tidy section layout |
| `ui/lib/widgets/prediction_list.dart` | Align result row spacing/typography with the rest of the screen |
| `ui/lib/widgets/image_preview.dart` | Consistent corner radius / border treatment |
| `ui/playwright.config.ts` | Set demo `viewport` + `video.size` to a standard iPhone screen |

### Cleanup checklist

- Flatten the `AppBar` — drop the `inversePrimary` fill in favor of a plain surface bar with no elevation
- Define a single spacing rhythm (reuse a small set of gaps instead of ad-hoc `8/16/20/24` values)
- Reuse one corner-radius value across preview, top-prediction, and error containers
- Keep one primary action (`Identify`) clearly dominant; ensure upload/URL read as secondary
- Confirm confidence percentages and labels stay easy to scan (US-06)
- Confirm the one-line tagline remains visible above the fold (US-05)
- Verify the layout still reads well at iPhone width (no overflow / clipped controls)

### Playwright iPhone dimensions

Swap the desktop viewport for a standard iPhone screen. Either reuse Playwright's
built-in device descriptor or set the dimensions directly:

```ts
import { defineConfig, devices } from '@playwright/test';

// iPhone 14 / 13 logical resolution: 390 × 844
export default defineConfig({
  use: {
    ...devices['iPhone 14'],
    baseURL: 'http://localhost:8080',
    viewport: { width: 390, height: 844 },
    video: { mode: 'on', size: { width: 390, height: 844 } },
  },
  // webServer entries unchanged
});
```

## Acceptance criteria

- [ ] Visual polish applied per the cleanup checklist; layout stays simple and flat (`AGENTS.md`)
- [ ] No behavior changes — upload, URL, loading, error, and results flows work as in T003
- [ ] All existing `Key`s and `Semantics` labels preserved; `npx playwright test` still passes
- [ ] Playwright demo `viewport` and `video.size` set to a standard iPhone screen (e.g. 390 × 844); resulting WebM is phone-shaped
- [ ] One-line app summary remains visible (US-05); confidence/labels stay easy to scan (US-06)
- [ ] `flutter analyze` reports no new warnings
- [ ] **Human:** reviewer confirms the UI looks cleaner and approves the result

## Out of scope

- New features, screens, or API/contract changes
- Renaming or removing test selectors (`Key`s / `Semantics`)
- New packages or design-system dependencies
- Mobile (iOS/Android) targets; animation or theming systems beyond light polish
