# T015 ‚Äî URL input component + decluttered Search relayout

**Status:** planned  
**Maps to:** US-13 (focused Search screen)  
**Depends on:** [T014 ‚Äî Image selection card](done/T014.md)

## Goal

Rebuild the Search screen around two clear, self-contained inputs: the `ImageSelectionCard` (T014) and a distinct URL input component, separated by a divider, with the single `Identify` button below. Drop the tagline header and remove the inline result from Search ‚Äî Search becomes a calm, focused entry point. (The result moves to its own tab in T016; until then it's acceptable for `_identify` to still hold results in state, just not render them on Search.)

## Approach

Extract the URL `TextField` into a small `UrlInputField` widget, then recompose `HomeScreen`'s body as: `ImageSelectionCard` ‚Üí `Divider` ("or") ‚Üí `UrlInputField` ‚Üí `Identify` button ‚Üí loading/error. Keep all existing picking/identify logic in `HomeScreen` (file pick, URL submit, `_identify`, auto-save) ‚Äî this is a layout + extraction change, not a behavior change to inference.

Wire the T014 card: `onPick` runs the existing `_pickImage`; the card receives `_bytes` (and `_previewUrl`/url for preview); typing a URL clears a picked file as it does today. Remove the `OutlinedButton.icon('Upload image')`, the "Selected: filename" line (the card now shows the selection), the `kAppTagline` header, and the inline `ResultView` from Search.

```
ui/lib/
  widgets/url_input_field.dart   # NEW ‚Äî labeled URL TextField (extracted)
  screens/home_screen.dart       # card + divider + URL + Identify; no tagline, no inline result
```

### Proposed files

| File | Purpose |
|------|---------|
| `ui/lib/widgets/url_input_field.dart` | `UrlInputField({ TextEditingController controller, bool enabled, ValueChanged<String>? onChanged, VoidCallback? onSubmitted })` ‚Äî keeps `Key('url-field')` |
| `ui/lib/screens/home_screen.dart` | Recompose body to card + divider + `UrlInputField` + `Identify`; remove tagline, upload button, "Selected:" line, inline `ResultView` |

### Layout (US-13)

- Order: `ImageSelectionCard` (T014) ‚Üí a divider with "or" (the existing `Row([Divider, 'or', Divider])` is fine) ‚Üí `UrlInputField` ‚Üí `Identify` (`FilledButton`, `Key('identify-button')`) ‚Üí loading indicator / error container.
- No `kAppTagline` header (the nav bar + card prompt already say what to do). `kAppTagline` may remain defined for reuse elsewhere (e.g. Settings/About) but is not rendered on Search.
- No `ResultView` on Search. The screen stays the same regardless of whether a prediction has been made.
- Keep the `maxWidth: 640` centered column, `DriftSpacing` rhythm, and flat styling.

### Preserve behavior & selectors

- Keep `_identify` (file vs URL branch), `_saveToHistory` / auto-save (US-08), loading (`Key('loading-indicator')`) and error (`Key('error-message')`) handling exactly as today.
- Keep `Key('url-field')` (now inside `UrlInputField`) and `Key('identify-button')`. The removed `Key('upload-button')` is replaced by the card's `Key('image-card')` (T014); T017 updates affected tests.
