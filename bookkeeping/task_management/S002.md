# S002 ‚Äî Prediction history & navigation

**Status:** planned  
**Maps to:** US-07 (bottom tab nav), US-08 (auto-save), US-09 (browse history), US-10 (reopen result), US-11 (manage/clear history)

## Goal

Turn the single-screen DriftID web app into a small multi-section app that remembers what the user has identified. Add a persistent bottom tab bar (Search / History / Settings), automatically save each successful identification to browser-local storage, and give users a History section to browse, reopen, and clear past results ‚Äî without ever re-running inference or uploading anything to a server.

## Conventions

- **Simple and flat** per `AGENTS.md` ‚Äî `NavigationBar`, plain lists, solid pastel fills, no gradients/heavy shadows. Reuse `DriftColors` / `DriftSpacing` / `kDriftRadius`.
- **Local-only storage** ‚Äî history lives in the browser (localStorage); nothing new is sent to the server (US-08).
- **Stable selectors** ‚Äî every new interactive element and section gets a `Key` and/or `Semantics` label so Playwright can target it (consistent with US-07 and the existing demo). Keep all existing S001 selectors intact.
- **One model** ‚Äî persist and render the existing `Prediction` shape; do not fork a second label format.

## Acceptance criteria (sprint)

- A real Settings feature set (theme switching, API config UI) ‚Äî Settings is a labeled placeholder tab only this sprint
