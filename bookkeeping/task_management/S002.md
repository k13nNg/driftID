# S002 — Prediction history & navigation

**Status:** planned  
**Maps to:** US-07 (bottom tab nav), US-08 (auto-save), US-09 (browse history), US-10 (reopen result), US-11 (manage/clear history)

## Goal

Turn the single-screen DriftID web app into a small multi-section app that remembers what the user has identified. Add a persistent bottom tab bar (Search / History / Settings), automatically save each successful identification to browser-local storage, and give users a History section to browse, reopen, and clear past results — without ever re-running inference or uploading anything to a server.

## Context

This sprint builds directly on the S001 web app:

- UI lives in `ui/` (Flutter Web), themed with the pastel palette + spacing tokens defined in [`done/T004.md`](done/T004.md) (`DriftColors`, `DriftSpacing`, `kDriftRadius` in `ui/lib/main.dart`).
- Inference goes through `ApiClient` (`ui/lib/services/api_client.dart`) → FastAPI (`done/T001.md`). History must reuse the existing `Prediction` model and **must not** call the API when reopening saved results (US-10).
- Playwright demo recording (WebM) was established in `done/T002.md` / `done/T003.md` and styled to an iPhone viewport in `done/T004.md`; T010 extends it.

Dependencies (build in order): **T005 → T006 → {T007 → T008, T009} → T010**.

## Architecture

```
AppShell (NavigationBar)            ui/lib/app_shell.dart
 ├── Search   → HomeScreen          (existing identify flow, T003/T004)
 │                 │ on success
 │                 ▼
 │            HistoryStore  ──────►  browser-local storage (localStorage)
 │                 ▲                     (T006; local-only, no server)
 ├── History  → HistoryScreen ──────┘   list (T007) → HistoryDetail (T008)
 │                 │                     manage / clear (T009)
 └── Settings → SettingsScreen       (minimal placeholder)

ResultView (shared widget)          image preview + top-k list
 used by both a fresh identification and a reopened saved result (T008)
```

## Conventions

- **Simple and flat** per `AGENTS.md` — `NavigationBar`, plain lists, solid pastel fills, no gradients/heavy shadows. Reuse `DriftColors` / `DriftSpacing` / `kDriftRadius`.
- **Local-only storage** — history lives in the browser (localStorage); nothing new is sent to the server (US-08).
- **Stable selectors** — every new interactive element and section gets a `Key` and/or `Semantics` label so Playwright can target it (consistent with US-07 and the existing demo). Keep all existing S001 selectors intact.
- **One model** — persist and render the existing `Prediction` shape; do not fork a second label format.

## Acceptance criteria (sprint)

- [ ] A persistent bottom tab bar exposes Search, History, and Settings; Search is the default landing section (T005, US-07)
- [ ] A successful identification is saved automatically and survives reload + new browser session (T006, US-08)
- [ ] History lists past entries most-recent-first with thumbnail, make/model, confidence, and time, plus an empty state (T007, US-09)
- [ ] Reopening an entry shows the full top-k result view and makes **no** API call (T008, US-10)
- [ ] A single entry can be deleted and the whole history cleared (with confirmation), updating storage immediately (T009, US-11)
- [ ] `npx playwright test` produces a WebM of the identify → History → reopen flow and the S001 demo still passes (T010)
- [ ] **Human:** reviewer has watched the history demo WebM and confirms the flow works and looks clean

## Out of scope

- Server-side persistence, accounts, or syncing across devices
- Exporting/sharing history, search/filter within history, tags or notes
- A real Settings feature set (theme switching, API config UI) — Settings is a labeled placeholder tab only this sprint
- Retraining, new model/classifier work, deployment (still out per the backlog)
- Mobile (iOS/Android) Flutter targets
