# S003 ‚Äî Settings & focused Search

**Status:** planned  
**Maps to:** US-12 (color theme), US-13 (focused Search screen), US-14 (dedicated result view)

## Goal

Give DriftID a real Settings feature and a calmer Search experience. Add a persisted color preference (Light / Dark / System) that themes the whole app, and redesign Search around two clear, self-contained inputs ‚Äî a single clickable image card and a separate URL field ‚Äî with the result moved off Search entirely into its own dedicated Result tab. After an identification, Search resets to a clean state and the app hands off to the Result tab; reopening a past identification from History lands in that same Result view.

## Context

This sprint builds on the S002 web app:

- UI lives in `ui/` (Flutter Web). The pastel palette + spacing tokens live in `ui/lib/main.dart` (`DriftColors`, `DriftSpacing`, `kDriftRadius`); today `MaterialApp` wires a single hardcoded **light** `ColorScheme` with no `darkTheme` / `themeMode`.
- Local persistence already has an established pattern: `HistoryStore` (`ui/lib/services/history_store.dart`) is a `ChangeNotifier` backed by `SharedPreferencesAsync` (‚Üí `localStorage`), constructed in `main.dart` and hydrated on startup. `SettingsStore` (T012) mirrors this exactly ‚Äî local-only, no server.
- The app shell (`ui/lib/app_shell.dart`) is an `IndexedStack` behind a `NavigationBar` with three fixed destinations (Search / History / Settings). T016 adds a fourth permanent destination (Result), so the order becomes Search / Result / History / Settings.
- Result presentation is already a shared widget, `ResultView` (`ui/lib/widgets/result_view.dart`), used by both Search and `HistoryDetailScreen`. T016 makes the Result **tab** the single host for it and retires `HistoryDetailScreen`.
- Inference still goes through `ApiClient`; reopening a saved result must continue to make **no** API call (US-10 carries over).

Dependencies (build in order): **T012 ‚Üí T013**, and **T014 ‚Üí T015 ‚Üí T016 ‚Üí T018**, then **T017** last. T014 can start in parallel with the theme track.

## Conventions

- **Simple and flat** per `AGENTS.md` ‚Äî Material 3 `NavigationBar`, segmented buttons, plain cards; solid fills, no gradients/heavy shadows. Reuse `DriftColors` / `DriftSpacing` / `kDriftRadius`.
- **Theme-aware colors** ‚Äî read colors from `Theme.of(context).colorScheme` rather than hardcoding `DriftColors.*` or literal hex in widgets, so dark mode renders correctly. Where a literal is unavoidable (e.g. the neutral image-preview surface), give it a light/dark variant.
- **Local-only storage** ‚Äî the theme preference lives in the browser (localStorage) alongside history; nothing new is sent to the server.
- **Stable selectors** ‚Äî every new interactive element/section gets a `Key` and/or `Semantics` label for Playwright. Preserve existing S001/S002 selectors that survive the redesign; where Search markup changes (e.g. the old `upload-button`), update the affected tests in T017.
- **One model / one result surface** ‚Äî keep persisting and rendering the existing `Prediction` shape; Search-fresh and History-reopened results share a single `ResultView` hosted by the Result tab.
