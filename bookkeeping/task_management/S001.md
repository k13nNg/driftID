# S001 — Setup Dependencies

**Status:** planned  
**Maps to:** US-01 (upload), US-02 (URL), US-03 (top-k), US-04 (real-time inference), US-05 (app at a glance), US-06 (readable labels)

## Goal

Stand up the product stack so inference UX can be built and demoed in a browser: a Python API for `Predictor`, a Flutter Web frontend, and Playwright for repeatable demo screen recordings (WebM).

## Tasks

| Task | Scope |
|------|--------|
| [T001 — FastAPI](T001.md) | HTTP API wrapping `Predictor` (upload, URL, top-k JSON) |
| [T002 — Flutter Web scaffold + Playwright smoke](T002.md) | Default counter app; Playwright increments 3×; WebM smoke test |
| [T003 — Flutter Web + Playwright](T003.md) | DriftID inference UI, API integration, demo video recordings |

## Architecture

```
Browser (Flutter Web)  --HTTP-->  FastAPI (T001)  -->  Predictor  -->  JSON
        ^
        |
   Playwright (T002 validates tooling, T003 records product demo)
```

## Flutter Web frontend

- Lives in a top-level `ui/` Flutter project (web target only for now)
- **T002:** stock counter template — proves Flutter Web + Playwright wiring
- **T003:** replace counter with DriftID UI; calls FastAPI from T001 (`POST /predict`, `POST /predict-url`)
- Simple, flat layout per project UI guidelines — one primary upload/URL flow, top-k results, loading and error states
- Widget `Key`s / `Semantics` labels on interactive elements so Playwright can target them reliably

## Playwright demo recordings

- Node + `@playwright/test` alongside the Flutter app (bootstrapped in T002, extended in T003)
- `recordVideo` enabled in config → **WebM** artifacts on each run
- **T002:** counter incremented 3 times — pipeline smoke test
- **T003:** happy path (upload or URL → loading → top-k results) for portfolio / README demos
- Not a full E2E test suite — optimized for consistent, re-recordable clips

## Acceptance criteria (story)

- [ ] FastAPI serves inference; frontend never imports Python/PyTorch (T001)
- [ ] Flutter Web + Playwright pipeline proven with counter smoke test (T002)
- [ ] Flutter Web app talks to the API with CORS (T003)
- [ ] Playwright demo script produces a WebM of the main inference flow (T003)
- [ ] UI satisfies US-01–US-06 acceptance criteria as implemented in T003

## Out of scope

- Mobile (iOS/Android) Flutter targets
- Authentication, rate limiting, production hosting
- Full CI E2E matrix — only demo recording scripts for now
