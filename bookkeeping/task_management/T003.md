### Flutter UI (US-01–US-06)

Keep layout **simple and flat** (see `AGENTS.md`):

- One-line app summary on the main screen (US-05)
- File picker for image upload with preview (US-01)
- Text field + submit for image URL (US-02)
- Loading indicator while awaiting API response (US-04)
- Top-k list ordered by confidence; top row emphasized (US-03)
- Labels from API displayed clearly; confidence as percentage (US-06)
- User-facing error strings for bad files, bad URLs, and API failures — no stack traces (US-01, US-02, US-04)

Add stable selectors for Playwright:


Default API base URL: `http://localhost:8000` (configurable for builds).

### Playwright demo setup

T002 already installs Playwright and configures video recording. Extend `playwright.config.ts` to also start the API:

```ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './demos',
  use: {
    baseURL: 'http://localhost:8080',
    viewport: { width: 1280, height: 720 },
    video: { mode: 'on', size: { width: 1280, height: 720 } },
  },
  webServer: [
    {
      command: 'uvicorn src.api.server:app --host 0.0.0.0 --port 8000',
      url: 'http://localhost:8000/health',
      cwd: '..',
      reuseExistingServer: true,
      timeout: 120_000,
    },
    {
      command: 'flutter run -d web-server --web-port 8080 --web-renderer html',
      url: 'http://localhost:8080',
      reuseExistingServer: true,
      timeout: 120_000,
    },
  ],
});
```

**`demos/record-demo.spec.ts`** — happy-path demo (adjust selectors to match UI):

```ts
import { test } from '@playwright/test';

test('upload and predict demo', async ({ page }) => {
  await page.goto('/');
  await page.getByLabel('Upload image').setInputFiles('demos/fixtures/sample-car.jpg');
  await page.getByRole('button', { name: 'Identify' }).click();
  await page.getByText(/confidence/i).first().waitFor({ state: 'visible' });
  await page.waitForTimeout(2000); // hold on results for the recording
});
```

Videos land under `ui/test-results/**/video.webm` after the test context closes.

For faster, stabler demos in CI or repeat runs, prefer serving a built app:

```bash
flutter build web --web-renderer html
npx serve build/web -l 8080
```

## Run (once implemented)

Terminal 1 — API (from repo root):

```bash
conda activate gpu-env
uvicorn src.api.server:app --host 0.0.0.0 --port 8000 --reload
```

Terminal 2 — Flutter Web:

```bash
cd ui
flutter run -d web-server --web-port 8080 --web-renderer html
```

Terminal 3 — record demo:

```bash
cd ui
npx playwright test
# WebM: test-results/.../video.webm
```

Or let Playwright start both servers via `webServer` in config (single command from `ui/`).

## API client usage (Flutter)

```dart
// File upload
final request = http.MultipartRequest(
  'POST',
  Uri.parse('$baseUrl/predict?k=5'),
)..files.add(await http.MultipartFile.fromPath('file', imagePath));

// URL
final response = await http.post(
  Uri.parse('$baseUrl/predict-url'),
  headers: {'Content-Type': 'application/json'},
  body: jsonEncode({'url': imageUrl, 'k': 5}),
);
```

Parse `predictions` list: `{ "class": "...", "confidence": 0.42 }`.
