# T011 — Pre-seeded history E2E paths

**Status:** done  
**Maps to:** US-08 (persistence), US-09 (browse), US-10 (reopen), US-11 (manage/clear)  
**Depends on:** [T010 — History demo recording + regression](done/T010.md) (Playwright setup), [T007](done/T007.md) / [T008](done/T008.md) / [T009](done/T009.md) (the screens under test)

## Goal

Add Playwright coverage for the history paths the current specs don't exercise — browsing a **populated** list, reopening a saved result **without** an API call, deleting a single entry, and clearing all history (confirm **and** cancel). Each test starts from a **pre-seeded user**: history is injected into browser-local storage before the app boots, so these flows run fast and deterministically without driving inference or depending on the backend at all.

Today's specs cover the happy path and empty/stub navigation only:

| Spec | Covers |
|------|--------|
| `record-demo.spec.ts` | upload → predict (US-01/03/04) |
| `record-history.spec.ts` | identify → History → reopen, from a fresh save (US-07–10) |
| `nav-history.spec.ts` | History tab **empty** state (US-09) |
| `nav-settings.spec.ts` | Settings stub (US-07) |

The management paths (US-11 delete/clear) and a multi-entry list (US-09 ordering) have **no** automated coverage, and reopen-without-inference (US-10) is only implied. T011 closes those gaps.

## Approach

Add one shared **seed helper** plus four focused specs. The helper writes a `HistoryEntry[]` JSON payload into `localStorage` via `page.addInitScript` (which runs *before* any page script), so by the time `DriftIDApp` calls `HistoryStore.load()` in `initState`, the entries already exist — exactly as if a real user had identified those cars in a prior session.

Crucially this needs **no** inference: the persisted `HistoryEntry` already carries the full top-k `predictions`, the image reference, and a timestamp (see `ui/lib/models/history_entry.dart`), which is everything History and the reopen view render.

```
ui/demos/

  history-browse.spec.ts   # NEW — populated list: ordering, labels, confidence, time (US-09)

  history-delete.spec.ts   # NEW — delete one entry; others remain (US-11)
  history-clear.spec.ts    # NEW — clear-all confirm + cancel (US-11)
```

### Seed payload (must mirror the persisted shape)

`HistoryStore` persists a JSON **array** of `HistoryEntry.toJson()` under a single `localStorage` key. The seed helper must reproduce that exact shape:

```ts
// one entry — matches HistoryEntry.toJson() in ui/lib/models/history_entry.dart
{
  id: '1718600000000000',                  // unique, sortable (microsecond-ish) string
  createdAt: '2026-06-16T22:30:00.000',    // ISO-8601; drives the relative-time label
  source: 'url',                           // 'url' (use a remote imageRef) or 'upload' (data: URL)
  imageRef: 'https://example.com/car.jpg', // remote URL for source:'url'
  predictions: [                           // full top-k; {class, confidence} pairs (US-10)
    { class: 'audi_a7-gen_2010_2014', confidence: 0.91 },
    { class: 'audi_a6-gen_2011_2015', confidence: 0.04 },
  ],
}
```

Notes for realistic, assertable fixtures:
- **Labels** are derived from `class` by `Prediction` (`make_model-gen_startYear_endYear` → e.g. `Audi A7`, subtitle `2010–2014`). Pick classes whose parsed titles are easy to assert (`audi_a7-gen_2010_2014`, `bmw_m3-gen_2014_2018`, `toyota_supra-gen_2019_2024`).
- **Relative time** comes from `createdAt` via `formatRelativeTime` (`history_tile.dart`): `< 60s` → "Just now", minutes → "Nm ago", hours → "Nh ago", 1 day → "Yesterday", `< 7d` → "Nd ago". Stagger `createdAt` (e.g. now, ‑5 min, ‑2 h) so the list shows distinct, ordered times.
- Prefer `source: 'url'` fixtures so the thumbnail/preview can load a real remote image without embedding a large base64 `data:` URL in the test. Include **one** `source: 'upload'` entry (a tiny 1×1 PNG `data:` URL) so the upload/data-URL branch is exercised too.
- The list renders **most-recent-first** (`HistoryStore` sorts desc by `createdAt`); assert that order regardless of array order in the seed.

### The `localStorage` key + value wrapper (verify once)

History is stored via `SharedPreferencesAsync` (key constant `HistoryStore.storageKey = 'driftid.history.v1'`). On web that maps to a `localStorage` entry, but the **exact** key (the `shared_preferences` `flutter.` prefix may apply) and any value wrapping that the async web backend adds are version-dependent and must not be guessed.

**Capture the real format once, then template it:**
1. `cd ui && flutter build web && python3 -m http.server 8080 --directory build/web`
2. Identify a car so one entry saves, then in DevTools → Application → Local Storage read the real key **and** its stored string.
3. Make `seed-history.ts` write that **same key** with the **same wrapper**, substituting our `HistoryEntry[]` JSON as the value.

If the captured key turns out to carry the `flutter.` prefix, seed `flutter.driftid.history.v1`; otherwise `driftid.history.v1`. Writing both (prefixed + unprefixed) is an acceptable belt-and-suspenders default if it avoids brittleness. Seeding **must** happen with `page.addInitScript` (before navigation) because `load()` reads storage at startup; a post-load `localStorage.setItem` would be missed until reload.

```ts
// ui/demos/helpers/seed-history.ts (illustrative)
import type { Page } from '@playwright/test';

export type SeedEntry = {
  id: string; createdAt: string; source: 'url' | 'upload';
  imageRef: string; predictions: { class: string; confidence: number }[];
};

export async function seedHistory(page: Page, entries: SeedEntry[]) {
  const payload = JSON.stringify(entries);
  await page.addInitScript(([key, value]) => {
    window.localStorage.setItem(key, value);
  }, ['driftid.history.v1', payload /* + flutter.-prefixed key / wrapper per capture above */]);
}
```

### Spec outlines

All specs reuse the semantics-enabling pattern from the existing specs (`waitForSelector('flt-semantics-placeholder')` → `dispatchEvent('click')`) and target the stable `Key`/`Semantics` already in the UI:

- History tabs → `getByRole('tab', { name: 'History' })`
- Entry tiles → `Semantics` button labeled `"<title>, <subtitle>, <when>"` → `getByRole('button', { name: /Audi A7/ })`
- Per-tile delete → `IconButton` tooltip `'Delete'`; "Clear all" → tooltip `'Clear all'`; confirm dialog buttons `'Clear'` / `'Cancel'`
- Snackbar → text `Removed from history`; empty state → text `No identifications yet`
- Reopen view → `getByRole('heading', { name: 'Saved result' })` + `getByText('Best match')`

**history-reopen — assert no inference (US-10):** attach a request guard before navigating and fail if the backend is touched while reopening:

```ts
page.on('request', (req) => {
  if (/\/predict(\b|-url|\?)/.test(req.url())) {
    throw new Error(`Unexpected inference call on reopen: ${req.url()}`);
  }
});
```

(The inference endpoints are `POST /predict` and `POST /predict-url`; see `ui/lib/services/api_client.dart`.)
