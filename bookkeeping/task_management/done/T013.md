# T013 — Theme control in Settings

**Status:** done  
**Maps to:** US-12 (choose a color theme)  
**Depends on:** [T012 — Settings store + theme modes](T012.md)

## Goal

Give the Settings tab a real control to choose the color theme — Light, Dark, or System — wired to the `SettingsStore` from T012 so the selection applies instantly and persists. Keep the existing About blurb; this is the first real Settings feature.

## Approach

Turn `SettingsScreen` into a small settings list. Add a "Theme" section with a flat segmented control (`SegmentedButton<ThemeMode>`) for Light / Dark / System whose selected value reads from `SettingsStore.themeMode` and whose `onSelectionChanged` calls `SettingsStore.setThemeMode(...)`. Rebuild the control on the store via `ListenableBuilder` so it reflects external changes (e.g. OS appearance under System). Keep the About text below it.

The store is already constructed in `main.dart` (T012); pass it down to `AppShell` → `SettingsScreen` the same way `HistoryStore` is threaded today.

```
ui/lib/
  screens/settings_screen.dart   # Theme segmented control + About; takes SettingsStore
  app_shell.dart                 # pass settingsStore into SettingsScreen
  main.dart                      # hand settingsStore to AppShell
```

### Proposed files

| File | Purpose |
|------|---------|
| `ui/lib/screens/settings_screen.dart` | "Theme" section with `SegmentedButton<ThemeMode>` (Light/Dark/System) bound to `SettingsStore`; keep the About blurb |
| `ui/lib/app_shell.dart` | Accept + forward `SettingsStore` to `SettingsScreen` |
| `ui/lib/main.dart` | Pass the hydrated `SettingsStore` into `AppShell` |

### Control behavior & selectors

- Segments, in order: **Light**, **Dark**, **System**, each with a label (and an icon, e.g. `light_mode` / `dark_mode` / `brightness_auto`); flat per `AGENTS.md`.
- Selecting a segment calls `setThemeMode` → the whole app re-themes immediately (T012 wiring) and the choice persists.
- Wrap in `ListenableBuilder(listenable: settingsStore)` so the selected segment stays in sync.
- Selectors for automation (T017): `Key('settings-theme')` on the control plus a stable handle per option — e.g. `Key('theme-light')`, `Key('theme-dark')`, `Key('theme-system')` (or `Semantics` labels) so Playwright can pick a mode and assert the applied brightness. Keep the existing `Key('settings-about')`.
