# T005 — App shell + bottom tab navigation

**Status:** done  
**Maps to:** US-07 (bottom tab navigation)  
**Depends on:** [T004 — UI cleanup](done/T004.md)

## Goal

Introduce a persistent bottom tab bar so DriftID has clear sections. Wrap the existing identify flow in an app shell with three tabs — **Search**, **History**, and **Settings** — where Search is the default landing section and continues to host the current upload/URL flow. This task delivers the navigation chrome and placeholder destinations; the History content is built in T007.

## Approach

Add an `AppShell` widget that owns a `NavigationBar` and an `IndexedStack` of the three destinations, and make it the `MaterialApp.home` (replacing `HomeScreen`). Using `IndexedStack` keeps each tab's state alive so switching tabs never triggers a full reload (US-07) and the Search flow keeps its picked image / results when the user pops over to History and back.

`HomeScreen` is reused as-is for the Search tab — do **not** change its `Key`s or `Semantics` labels (the S001 demo and T010 depend on them). History and Settings start as minimal placeholder screens; T007 fills in History.

```
ui/lib/
  app_shell.dart            # NEW — NavigationBar + IndexedStack
  main.dart                 # home: const AppShell()  (was HomeScreen)
  screens/
    home_screen.dart        # unchanged — Search tab body
    history_screen.dart     # NEW — placeholder this task (filled in T007)
    settings_screen.dart    # NEW — minimal placeholder
```

### Proposed files

| File | Purpose |
|------|---------|
| `ui/lib/app_shell.dart` | `StatefulWidget` holding selected index; `NavigationBar` + `IndexedStack([HomeScreen, HistoryScreen, SettingsScreen])` |
| `ui/lib/main.dart` | Point `home:` at `AppShell`; keep theme/palette untouched |
| `ui/lib/screens/history_screen.dart` | Placeholder body (`'No history yet'`) — real content in T007 |
| `ui/lib/screens/settings_screen.dart` | Minimal placeholder (app name + one-line about) |

### Navigation behavior

- Default selected index is **0 → Search**, so a fresh load lands on the identify flow (US-07).
- `NavigationBar` destinations, in order: Search (`Icons.search`), History (`Icons.history`), Settings (`Icons.settings`).
- Selecting a destination updates the index via `setState`; `IndexedStack` swaps the visible child with no route push / page reload.
- Style flat per `AGENTS.md`: default Material 3 `NavigationBar` over the existing `ColorScheme`, no elevation tricks or custom animation.
- The `AppBar` title can move into the shell (or stay per-screen) — keep one clear title per section; Search keeps `'DriftID'`.

### Selectors for automation (US-07, T010)

Give each destination a stable, labeled handle:

```dart
NavigationDestination(
  icon: Semantics(label: 'History tab', child: const Icon(Icons.history)),
  label: 'History',
),
```

Add `Key`s on the bar / destinations as needed: `Key('nav-search')`, `Key('nav-history')`, `Key('nav-settings')`. Tab labels must surface as accessible names so Playwright can do `getByRole('button'|'tab', { name: 'History' })` once Flutter semantics are enabled.
