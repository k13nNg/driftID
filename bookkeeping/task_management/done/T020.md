# T020 — Real car images in default test data

**Status:** done  
**Maps to:** US-09 (browse history) — verification fidelity  
**Depends on:** [T018 — Live URL image preview + shared preview component](done/T018.md)

## Goal

Make the default seeded test data show a real car image for every entry. Today the History/Result fixtures use placeholder refs — `REMOTE_IMAGE_REF` (`https://example.com/car.jpg`, which never resolves) and `TINY_PNG_DATA_URL` (a 1×1 transparent pixel) — so seeded History thumbnails and the reopened Result view render as blank or broken images. After this task, no seeded entry is "blank-car": each one renders a visible car photo, so the recorded demos and regression specs reflect real usage.

## Approach

The fixtures all flow through `seedHistory(...)` in `ui/demos/helpers/seed-history.ts`, which sets each entry's `imageRef`. Because the seeded entries hydrate `HistoryStore` *before navigation* and never hit the network in CI, the image must be self-contained and offline-safe — i.e. a `data:` URL, not a remote URL. Replace the two placeholder constants with one or more **real car photo `data:` URLs** (small, web-optimized JPEG/PNG encoded as base64) and default every seeded entry to one of them.

Keep at least one `source: 'url'` and one `source: 'upload'` fixture so both branches stay covered, but point both at embedded car images so neither renders blank. The existing `ui/demos/fixtures/sample-car.jpg` (used by the upload picker specs) is the natural source image to encode.

```
ui/demos/
  fixtures/
    sample-car.jpg          # existing upload fixture (source of truth for the photo)
    seed-cars.ts            # NEW (optional) — base64 data URLs generated from fixture images
  helpers/
    seed-history.ts         # UPDATE — replace placeholder refs with real car data URLs
  history-browse.spec.ts    # UPDATE — assert thumbnails render (no blank/broken image)
  history-reopen.spec.ts    # UPDATE — reopened Result shows the real image
  (other history specs)     # UPDATE only where the placeholder refs were referenced
```

### Proposed files

| File | Purpose |
|------|---------|
| `ui/demos/fixtures/seed-cars.ts` | Real car images as base64 `data:` URLs (generated from the fixture JPEGs), exported for seeding |
| `ui/demos/helpers/seed-history.ts` | Default seeded entries to the real car `data:` URLs; retire / repoint `REMOTE_IMAGE_REF` and `TINY_PNG_DATA_URL` |

### Details

- Keep images small (encode a downscaled version of `sample-car.jpg`, e.g. ≤256 px on the long edge) so the inlined base64 doesn't bloat the specs.
- Preserve the `source: 'url' | 'upload'` coverage and the `SeedEntry` shape; only the default `imageRef` values change.
- Update any spec that imported `TINY_PNG_DATA_URL` / `REMOTE_IMAGE_REF` directly so it still compiles, and tighten the relevant assertions to confirm an image actually paints (e.g. the shared preview renders, no broken-image fallback).

## Run

```bash
cd ui
flutter build web
npx playwright test --project=check   # regression gate; seeded history shows real cars
npx playwright test --project=record  # re-record affected History demo(s)
```

- [x] `npx playwright test --project=check` passes with the new fixtures

- [ ] **Human:** reviewer confirms the re-recorded History demo shows real car thumbnails throughout
