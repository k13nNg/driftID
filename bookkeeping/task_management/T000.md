# S001 — Dev container dependencies

**Status:** done  
**Maps to:** — (tooling; no product user stories)  
**Depends on:** nothing

## Goal

Install all Sprint 1 dependencies into the dev container so `gpu-env`, FastAPI, Flutter Web, and Playwright are available after rebuild — no manual one-off installs on each machine.

## Approach

Extend `.devcontainer/` so image build (or `postCreateCommand`) provisions the full stack. Other tasks (T001–T003) assume this environment is ready.

```
.devcontainer/
  Dockerfile           # Flutter SDK, Node.js, conda env
  environment.yml      # Python deps incl. FastAPI stack
  devcontainer.json    # postCreateCommand for Playwright browsers, flutter doctor
```

### Python (conda `gpu-env`)

Add API dependencies to `environment.yml` (used at image build):

```yaml
dependencies:
  # ... existing pytorch, timm, etc.
  - fastapi
  - uvicorn
  - python-multipart
```

Existing `postCreateCommand` smoke import (`torch`, `timm`, `faiss`) should still pass after rebuild.

### Flutter + Node

Install in `Dockerfile` (or devcontainer features) so `flutter` and `node`/`npm` are on `PATH` for the `vscode` user:

- Flutter SDK — stable channel, web target enabled (`flutter config --enable-web`)
- Node.js LTS — for `@playwright/test` under `ui/`

### Post-create (optional)

If browser binaries are too heavy for the image layer, run in `devcontainer.json` `postCreateCommand`:

```bash
cd ui && flutter pub get && npm ci && npx playwright install chromium
```

(Exact command can wait until `ui/package.json` exists in T002; until then, document the intent and install Flutter/Node at build time.)

## Run (once implemented)

Rebuild the dev container (VS Code: **Dev Containers: Rebuild Container**), then verify:

```bash
conda activate gpu-env
python -c "import fastapi, uvicorn; print('API deps OK')"
flutter --version
node --version
```

## Acceptance criteria

- [x] `.devcontainer/environment.yml` includes FastAPI, uvicorn, and python-multipart
- [x] Dev container image provides `flutter` on PATH with web support
- [x] Dev container image provides `node` and `npm` on PATH
- [x] Rebuild succeeds; existing `postCreateCommand` conda smoke check still passes
- [x] No manual `pip install` / SDK downloads required after a fresh container start (Playwright browsers may use `postCreateCommand` once `ui/` exists)

## Out of scope

- Flutter app scaffold or Playwright specs (see T002)
