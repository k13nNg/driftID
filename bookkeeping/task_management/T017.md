# T017 — Demo recording + E2E regression

**Status:** planned  
**Maps to:** US-12 (theme), US-13 (focused Search), US-14 (Result tab) — verification  
**Depends on:** [T013 — Theme control in Settings](T013.md), [T016 — Result tab + shared result surface](T016.md)

## Goal

Prove the S003 changes end-to-end with Playwright: record a WebM showing the theme switch and the new Search → Result flow, and update the existing demo/regression suite so it passes against the redesigned Search (single image card, no "Upload image" button) and the Result tab (results no longer inline on Search).

## Approach

Add a new recorded demo spec under `ui/demos/` for S003 and fix the specs whose selectors/assumptions the redesign breaks. The biggest breakages are: the image card replaces the `Upload image` button (`record-demo.spec.ts`, `record-history.spec.ts` pick via that button), and the result is now on the Result tab rather than inline on Search.

```
ui/demos/
  record-settings-theme.spec.ts   # NEW — switch Light/Dark/System, hold frames (recording)
  record-demo.spec.ts             # UPDATE — pick via image card; assert result on Result tab
  record-history.spec.ts          # UPDATE — same picker change; reopen lands on Result tab
  history-reopen.spec.ts          # UPDATE — reopen asserts the Result tab/view, no API call
  (other history specs)           # UPDATE only where picker/result assumptions changed
```

### Test updates

- **Picker:** replace `getByRole('button', { name: 'Upload image' })` + `filechooser` with a click on the image card (`getByKey`/role for `Key('image-card')`, or its `Semantics` label e.g. "Add a car photo") that still triggers the `filechooser` event and `setFiles('demos/fixtures/sample-car.jpg')`.
- **Result location:** after `Identify`, assert the app is on the Result tab and that `Best match` is visible there (instead of inline on Search). Optionally assert Search is clean after switching back.
- **Reopen (US-10):** `history-reopen.spec.ts` should assert the reopened entry appears in the Result view marked saved (`Key('saved-result')`) and that **no** `/predict*` request fires (keep/extend the existing no-API assertion).
- Keep semantics-enable boilerplate (`flt-semantics-placeholder` click) and the iPhone-viewport recording setup from `playwright.config.ts`.

### New recorded demos

- `record-settings-theme.spec.ts`: open Settings → tap **Dark** (app darkens) → tap **System**/**Light**, holding frames so the WebM clearly shows the theme changing. Uses `Key('settings-theme')` / `Key('theme-dark')` etc. from T013.
- Extend or add a recorded flow for **identify → Result tab → back to clean Search** so the new UX is captured on video (can be folded into the updated `record-demo.spec.ts`).

## Run

```bash
cd ui
flutter build web         # or run the dev server the config expects
npx playwright test       # full suite green against the redesign
# WebMs land in ui/test-results/**/video.webm
```

## Acceptance criteria

- [ ] `npx playwright test` passes against the redesigned Search (image card, no upload button) and Result tab
- [ ] A WebM captures the theme switch (Light/Dark/System) (US-12)
- [ ] A WebM captures identify → Result tab (full top-k) → clean Search (US-13/US-14)
- [ ] The reopen-from-history path asserts the Result view marked saved with **no** API call (US-10/US-14)
- [ ] Specs that relied on the removed `Upload image` button / inline result are updated to the new selectors
- [ ] **Human:** reviewer has watched the S003 WebM(s) and confirms theming + the focused Search/Result flow work and look clean

## Out of scope

- New product behavior (all owned by T012–T016)
- Non-web (mobile) test targets
- Visual-regression/pixel snapshots beyond the recorded WebM
