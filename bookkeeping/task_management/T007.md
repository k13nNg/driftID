# T007 — Browse past identifications

**Status:** planned  
**Maps to:** US-09 (browse past identifications)  
**Depends on:** [T005 — App shell + bottom tab navigation](done/T005.md), [T006 — Local history store + auto-save](done/T006.md)

## Goal

Fill in the History tab with a scannable list of saved identifications, ordered most-recent-first, each showing a thumbnail, the top make/model, its confidence, and when it was identified. Show a helpful empty state when there is no history yet.

## Approach

Replace the T005 placeholder `HistoryScreen` with a list driven by `HistoryStore` (rebuild on `notifyListeners` via `AnimatedBuilder`/`ListenableBuilder`). Render one tappable row per `HistoryEntry`; tapping is wired in T008. Use the shared palette/spacing tokens and keep the row visually flat.

```
ui/lib/
  screens/history_screen.dart       # list + empty state (replaces T005 placeholder)
  widgets/history_tile.dart         # NEW — one row: thumbnail + label + confidence + time
  widgets/history_empty.dart        # NEW — empty-state guidance
```

### Proposed files

| File | Purpose |
|------|---------|
| `ui/lib/screens/history_screen.dart` | Subscribe to `HistoryStore`; `ListView` of `HistoryTile` or empty state |
| `ui/lib/widgets/history_tile.dart` | Thumbnail (leading) + title/subtitle + confidence + relative time |
| `ui/lib/widgets/history_empty.dart` | Icon + message explaining how entries get added |

### List behavior (US-09)

- Reachable from the History tab (the section built in T005).
- Entries ordered **most-recent-first** (rely on `HistoryStore.entries` ordering from T006).
- Each tile shows:
  - **Thumbnail** — `Image.memory` (decoded from the stored data URL) for uploads, `Image.network` for URL entries; fixed small size with `kDriftRadius` corners and a graceful fallback (reuse the placeholder treatment from `ImagePreview`).
  - **Top make/model** — `entry.predictions.first.title` (+ `subtitle` years when present), via the existing `Prediction` getters (US-06 carries over).
  - **Confidence** — `entry.predictions.first.confidencePercent`.
  - **When identified** — relative time (e.g. "2h ago", "Yesterday") derived from `createdAt`; compute with a small local helper (no new dependency required) or `intl` if already available.
- `ListView.separated` with a flat divider; reuse `DriftSpacing` for padding. No gradients/shadows.

### Empty state (US-09)

When `entries` is empty, show `HistoryEmpty`: a car/clock icon, a line like *"No identifications yet"*, and one sentence — *"Identify a car on the Search tab and it'll show up here automatically."* — so users know how entries appear.

### Selectors for automation (T010)

- `Key('history-list')` on the `ListView`; `Key('history-empty')` on the empty state.
- Each tile: `Key('history-tile-<id>')` and a `Semantics` label combining make/model + time so Playwright can find a specific entry.
