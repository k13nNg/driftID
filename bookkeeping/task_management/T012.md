# T012 — Settings store + theme modes (Light / Dark / System)

**Status:** planned  
**Maps to:** US-12 (choose a color theme)  
**Depends on:** nothing (builds on the S002 app)

## Goal

Make DriftID theme-able. Add a persisted color preference with three values — Light, Dark, and System — and wire it through `MaterialApp` so the whole app re-themes instantly when it changes and restores the saved choice on reload / new session. This task delivers the store, the light + dark `ColorScheme`s, and the wiring; the Settings UI that drives it is T013.

## Approach

Add a `SettingsStore` `ChangeNotifier` that mirrors `HistoryStore`: backed by `SharedPreferencesAsync` (→ `localStorage`), constructed in `main.dart`, hydrated once on startup, persisted on change, and degrading gracefully to a default when storage is missing/corrupt. Persist the preference as a stable string (`'light' | 'dark' | 'system'`) mapped to/from `ThemeMode`.

Promote the current hardcoded light `ColorScheme` into a named `lightColorScheme` and add a sibling `darkColorScheme` built from a dark variant of the palette (pastel accents on dark surfaces, light ink). Wrap `MaterialApp` in a listener on `SettingsStore` so `theme` / `darkTheme` / `themeMode` rebuild when the preference changes.

Then audit widgets for hardcoded colors that would break in dark mode (notably `ImagePreview`'s neutral grey surface/border and any literal hex) and route them through `Theme.of(context).colorScheme` or give them light/dark variants.

```
ui/lib/
  services/settings_store.dart   # NEW — ChangeNotifier<ThemeMode>, persisted
  main.dart                      # lightColorScheme + darkColorScheme; ListenableBuilder → MaterialApp themeMode
  widgets/image_preview.dart     # theme-aware preview surface (no fixed light grey in dark mode)
```

### Proposed files

| File | Purpose |
|------|---------|
| `ui/lib/services/settings_store.dart` | `SettingsStore extends ChangeNotifier`; `ThemeMode get themeMode`, `Future<void> setThemeMode(ThemeMode)`, `Future<void> load()`; `SharedPreferencesAsync` backed |
| `ui/lib/main.dart` | Construct + hydrate `SettingsStore` (like `HistoryStore`); define `lightColorScheme`/`darkColorScheme` + matching `ThemeData`; rebuild `MaterialApp` on the store |
| `ui/lib/widgets/image_preview.dart` | Replace fixed `0xFFEDED..` surface/border with theme-derived colors (or light/dark variants) so previews read correctly in both modes |

### Store behavior

- Storage key: `'driftid.settings.theme_mode.v1'`; value one of `light` / `dark` / `system`.
- Default when absent/unreadable: **System** (`ThemeMode.system`).
- `load()` is called once on startup (alongside `HistoryStore.load()`); `setThemeMode` updates in-memory + notifies first, then persists best-effort (storage failures never throw into the UI).
- Injectable for tests (`SettingsStore({SharedPreferencesAsync? prefs})`), same lazy-plugin pattern as `HistoryStore`.

### Dark palette

- Add a dark `ColorScheme` (`brightness: Brightness.dark`) keeping the DriftID identity: pastel accents (`thistle` / `skyBlue` / `babyPink`) as primary/secondary/tertiary over dark surfaces, with a light "ink" for `onSurface`. Pick a dark surface (e.g. near-`#1C1A22`) and a readable on-surface; ensure contrast for prediction text (US-06 carries over).
- Keep it flat (no gradients/elevation tricks); both `AppBarTheme` and `NavigationBar` should derive from the active scheme.

## Acceptance criteria

- [ ] A `SettingsStore` persists the theme preference to `localStorage` and restores it on reload + new session (US-12)
- [ ] `MaterialApp` exposes both a light and a dark theme and follows `SettingsStore.themeMode` (`light` / `dark` / `system`) (US-12)
- [ ] Changing the preference re-themes the entire app with no reload (verified once T013 lands; until then via default/system) (US-12)
- [ ] Dark mode is legible across Search, History, Result, and Settings — no hardcoded light-only colors leak through (e.g. image preview surface) (US-06/US-12)
- [ ] `flutter analyze` reports no new warnings
- [ ] Existing S001/S002 selectors and behavior are unchanged by the theming work

## Out of scope

- The Settings UI control itself (T013)
- Accent-color pickers, per-section themes, font scaling, or design-token theming beyond Light/Dark
- Animating the theme transition
